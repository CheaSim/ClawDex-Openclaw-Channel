# Security Policy

## Supported versions

The latest version on the default branch is the supported release target for security fixes.

## Reporting a vulnerability

If you find a security issue in this plugin, open a GitHub issue in this repository with:

1. A clear description of the vulnerability.
2. Reproduction steps or a proof of concept.
3. The affected version and environment details.

Please avoid posting secrets, access tokens, or private user data in the report.
